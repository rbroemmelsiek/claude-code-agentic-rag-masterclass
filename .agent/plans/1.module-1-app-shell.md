Multiple interconnected components (auth, API, UI, OpenAI integration, streaming) but achievable in sequential tasks. Each task builds on the last.

## Overview

Build the foundation: authentication, chat interface, and OpenAI Responses API integration with LangSmith tracing.

OpenAI's Responses API provides "managed RAG" вЂ” it handles threading, memory, and file_search internally. This is a black box: convenient but opaque. Module 2 replaces it with the standard Chat Completions API for provider flexibility.

### Task 3: Database Schema (Supabase Migration)
**Files:** `supabase/migrations/001_initial_schema.sql`

Create SQL migration with:

**Tables:**
- `threads` вЂ” `id` (uuid PK), `user_id` (uuid, references auth.users), `openai_thread_id` (text), `title` (text), `created_at` (timestamptz), `updated_at` (timestamptz)
- `messages` вЂ” `id` (uuid PK), `thread_id` (uuid FK в†’ threads), `user_id` (uuid, references auth.users), `openai_message_id` (text), `role` (text, check: 'user'/'assistant'), `content` (text), `created_at` (timestamptz)

**RLS policies (both tables):**
- `SELECT` вЂ” `auth.uid() = user_id`
- `INSERT` вЂ” `auth.uid() = user_id`
- `UPDATE` вЂ” `auth.uid() = user_id` (threads only, for title updates)
- `DELETE` вЂ” `auth.uid() = user_id`

**Indexes:** on `threads.user_id`, `messages.thread_id`, `messages.created_at`

**Validation:** Run migration in Supabase SQL Editor. Verify tables exist with RLS enabled (shield icon). Attempt cross-user access вЂ” should be blocked.

### Task 8: OpenAI Responses API Service

### Task 10: Chat API with SSE Streaming
**Files:** `backend/app/routers/chat.py`

Endpoints (all require auth):
- `GET /threads/{thread_id}/messages` вЂ” fetch messages for a thread from DB
- `POST /threads/{thread_id}/messages` вЂ” accept `{"content": "..."}`, then:
  1. Save user message to DB
  2. Add message to OpenAI thread
  3. Run assistant, stream response via SSE (`text/event-stream`)
  4. On completion, save assistant message to DB
  5. Auto-generate thread title from first message (update DB)

SSE format:
```
data: {"type": "delta", "content": "chunk of text"}
data: {"type": "done", "message": {full message object}}
```

**Validation:** Use curl or a quick script to POST a message and observe SSE chunks arriving.
