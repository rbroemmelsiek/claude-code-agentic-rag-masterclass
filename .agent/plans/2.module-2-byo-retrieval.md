# Module 2: BYO Retrieval + Memory

**Complexity:** 🔴 Complex (5 phases across 3 stages, ~25 files touched)

**Status:** Not started — track progress in [PROGRESS.md](../../PROGRESS.md)

## Overview

Replace OpenAI's managed Responses API + `file_search` with:

1. **Provider abstraction** — swap to standard ChatCompletions API (works with OpenRouter, Ollama, LM Studio, any OpenAI-compatible endpoint)
2. **Stateless chat memory** — load and send full message history on every call (no more OpenAI-managed threads)
3. **Custom ingestion pipeline** — upload → chunk → embed → pgvector
4. **Retrieval tool** — vector search via standard tool calling loop
5. **Ingestion UI** — drag-and-drop upload with Supabase Realtime status updates

This is the "Option A: Replace" path from the PRD — Responses API code is removed entirely.

```python
async def astream_chat_response(
    messages: list[dict],
    tools: list[dict] | None = None,
) -> AsyncGenerator[dict, None]:
    request_kwargs = {
        "model": settings.llm_model,
        "messages": [{"role": "system", "content": SYSTEM_PROMPT}, *messages],
        "stream": True,
    }
    if tools:
        request_kwargs["tools"] = tools

    stream = await async_client.chat.completions.create(**request_kwargs)

    full_response = ""
    tool_calls_buffer = {}

    async for chunk in stream:
        delta = chunk.choices[0].delta if chunk.choices else None
        finish_reason = chunk.choices[0].finish_reason if chunk.choices else None

        if delta and delta.content:
            full_response += delta.content
            yield {"type": "text_delta", "content": delta.content}

        if delta and delta.tool_calls:
            for tc in delta.tool_calls:
                idx = tc.index
                if idx not in tool_calls_buffer:
                    tool_calls_buffer[idx] = {
                        "id": tc.id, "name": tc.function.name, "arguments": ""
                    }
                if tc.function and tc.function.arguments:
                    tool_calls_buffer[idx]["arguments"] += tc.function.arguments

        if finish_reason == "tool_calls":
            yield {"type": "tool_calls", "tool_calls": list(tool_calls_buffer.values())}

        if finish_reason == "stop":
            yield {"type": "response_completed", "content": full_response}
```

## Phase 2: Database Schema (pgvector + Documents + Chunks) — Stage A

### 2a. New migration: `supabase/migrations/20260123000000_module2_documents.sql`

```sql
CREATE EXTENSION IF NOT EXISTS vector;

CREATE TABLE documents (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
    filename TEXT NOT NULL,
    file_type TEXT NOT NULL,
    file_size INTEGER NOT NULL,
    storage_path TEXT NOT NULL,
    status TEXT NOT NULL DEFAULT 'pending'
        CHECK (status IN ('pending', 'processing', 'completed', 'failed')),
    error_message TEXT,
    chunk_count INTEGER DEFAULT 0,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE chunks (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    document_id UUID NOT NULL REFERENCES documents(id) ON DELETE CASCADE,
    user_id UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
    content TEXT NOT NULL,
    chunk_index INTEGER NOT NULL,
    embedding vector(1536),
    metadata JSONB DEFAULT '{}',
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Indexes
CREATE INDEX idx_documents_user_id ON documents(user_id);
CREATE INDEX idx_documents_status ON documents(status);
CREATE INDEX idx_chunks_document_id ON chunks(document_id);
CREATE INDEX idx_chunks_user_id ON chunks(user_id);
CREATE INDEX idx_chunks_embedding ON chunks
    USING ivfflat (embedding vector_cosine_ops) WITH (lists = 100);

-- RLS
ALTER TABLE documents ENABLE ROW LEVEL SECURITY;
ALTER TABLE chunks ENABLE ROW LEVEL SECURITY;

CREATE POLICY "users_own_documents" ON documents FOR ALL USING (auth.uid() = user_id);
CREATE POLICY "users_own_chunks" ON chunks FOR ALL USING (auth.uid() = user_id);

-- Realtime (for ingestion status updates in the UI)
ALTER PUBLICATION supabase_realtime ADD TABLE documents;

-- Vector search function
CREATE OR REPLACE FUNCTION match_chunks(
    query_embedding vector(1536),
    match_threshold float,
    match_count int,
    p_user_id uuid
) RETURNS TABLE (
    id uuid, document_id uuid, content text,
    chunk_index int, metadata jsonb, similarity float
) LANGUAGE plpgsql AS $$
BEGIN
    RETURN QUERY
    SELECT c.id, c.document_id, c.content, c.chunk_index, c.metadata,
           1 - (c.embedding <=> query_embedding) AS similarity
    FROM chunks c
    WHERE c.user_id = p_user_id
      AND 1 - (c.embedding <=> query_embedding) > match_threshold
    ORDER BY c.embedding <=> query_embedding
    LIMIT match_count;
END;
$$;
```

```python
async def get_embeddings(texts: list[str]) -> list[list[float]]:
    response = await embedding_async_client.embeddings.create(
        model=settings.embedding_model,
        input=texts,
        dimensions=settings.embedding_dimensions,
    )
    return [item.embedding for item in response.data]
```

### `chunking_service.py`

Pure Python recursive splitter — no external library:

- Separators: `["\n\n", "\n", ". ", " "]` tried in order
- `chunk_size=1000` characters, `chunk_overlap=200`
- Returns `list[str]`

### `ingestion_service.py`

`async def ingest_document(document_id: str, user_id: str, storage_path: str)`:

1. Update `documents.status` → `"processing"` + `updated_at`
2. Download file bytes from Supabase Storage (`storage_path`)
3. Extract text:
   - `.txt`: decode UTF-8
   - `.md`: decode UTF-8 (Module 2 only — PDF/DOCX in Module 5)
   - Unsupported → raise error
4. Chunk text via `chunking_service`
5. Batch-embed all chunks (batch size 100 to avoid rate limits)
6. Insert rows into `chunks` table with embeddings
7. Update `documents.status` → `"completed"`, set `chunk_count`
8. On any exception: update `documents.status` → `"failed"`, set `error_message`

### `routers/documents.py`

```
POST   /documents/upload       — Upload file, store to Supabase Storage, create DB record,
                                 trigger background ingestion via FastAPI BackgroundTasks
GET    /documents              — List user's documents (ordered by created_at desc)
DELETE /documents/{id}         — Delete document record + storage file (chunks cascade)
```

Request for upload: `multipart/form-data` with `file` field.

Response schemas: `DocumentResponse` (id, filename, file_type, file_size, status, chunk_count, created_at, updated_at, error_message).

Allowed file types for upload validation: `.txt`, `.md`.

### `retrieval_service.py`

```python
async def search_documents(query: str, user_id: str, top_k: int = 5, threshold: float = 0.7) -> list[dict]:
    query_embedding = await get_embeddings([query])
    result = supabase.rpc("match_chunks", {
        "query_embedding": query_embedding[0],
        "match_threshold": threshold,
        "match_count": top_k,
        "p_user_id": user_id,
    }).execute()
    return result.data
```

### `llm_service.py` additions

Add `RAG_TOOLS` constant:

```python
RAG_TOOLS = [{
    "type": "function",
    "function": {
        "name": "search_documents",
        "description": "Search the user's uploaded documents for relevant information.",
        "parameters": {
            "type": "object",
            "properties": {
                "query": {"type": "string", "description": "Search query"}
            },
            "required": ["query"]
        }
    }
}]
```

Add helper `def has_user_documents(user_id: str) -> bool` — checks if user has any `completed` documents in DB. Used by chat router to decide whether to pass tools.

### `tool_executor.py`

```python
async def execute_tool(tool_name: str, arguments: dict, user_id: str) -> str:
    if tool_name == "search_documents":
        results = await retrieval_service.search_documents(
            query=arguments["query"], user_id=user_id
        )
        if not results:
            return "No relevant documents found."
        return "\n\n".join(
            f"[Chunk {r['chunk_index']} | similarity {r['similarity']:.2f}]\n{r['content']}"
            for r in results
        )
    raise ValueError(f"Unknown tool: {tool_name}")
```

### `routers/chat.py` — tool-calling loop

```
1. Load prior messages from DB
2. Build messages list (user + history)
3. Determine tools: if user has completed documents → pass RAG_TOOLS, else None
4. Call astream_chat_response(messages, tools)
5. Loop:
   a. If tool_calls event:
      - Execute each tool via tool_executor
      - Append tool result messages to messages list
      - Call astream_chat_response again (max 3 rounds)
   b. If text_delta: yield SSE text_delta event
   c. If response_completed: save assistant message, yield SSE done event, break
```
